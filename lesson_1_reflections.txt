How did viewing a diff between two versions of a file help you see the bug that
was introduced?

Порівнюючи два файла, а саме рядки котрі були змінені (видалені чи додані) можна зрозуміти в чому проблема, так як не потрібно продивлятися два файла повністю, а можна зосередити увагу саме на змінах, котрі привели до багу.

How could having easy access to the entire history of a file make you a more
efficient programmer in the long term?

Це дає змогу побачити зміни, котрі вносилися (по-суті це бекап). В разі критичної помилки, завжди можна повернутися до версії котра працювала або ж побачити на якому етапі була зроблена помилка.

What do you think are the pros and cons of manually choosing when to create a
commit, like you do in Git, vs having versions automatically saved, like Google
docs does?

 Основним плюсом автоматичних комітів в Google Docs є те, що зміни автоматично зберігаються і можна бути на 100: впевненим, що нвчого не втрачено не загублено і завжди можна повернутися до ранішої версії чи зміни. Мінусом є невиправдана кількість цих збережень і відсутність логічного звязку, тобто для пошуку потрібної речі чи частини клду доведеться передевитися велеченку кількість версій.

 Основим плюсом створення комітів в Git є те, що вони ставорюються самостійно, тобто, зберігаються тільки логічно закінчені чи логічно обосновані зміни з повідомленнями, за котрими досить зручно відслідкуваи логіку. В той час це є і мінусом, тобто можуть бути занадто часті коміти, ніяким чином не обгрунтовані й історія буде просто переповнена ними, або занадто рідкі коміти й розібратися в велечезному кусков коду і змінах буде дуже тяжко й займе більше часу.

Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

Тому що це дає змогу відслідкувати логіку змін, немає сенсу робити три коміти котрі спярмовані на вирішення однієї проблеми, особливо якщо це стосуєть різних файлів. Наприклад створення кнопки буде вимагати змін в html, css і js, для відслідковування івентів наприклад. Тому логічніше якщо це убде одним комітом, а не 3.

How can you use the commands git log and git diff to view the history of files?

За допомогою git log можна проглянути всю історію комітів і за коміт меседжем знайти потрібний коміт, в той час як git diff дає змогу порівняти 2 коміти й побачити що змінилося, які рядки коду були змінені.

How might using version control make you more confident to make changes that
could break something?

Використання контролю версій дозволяє вносити будь які зміни, так як завжди можна бути впевненим, що можна повернутися до стану, коли програма чи под працюваав коректно, тому можна експерементувти.
